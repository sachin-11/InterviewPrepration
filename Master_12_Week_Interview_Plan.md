## 🗓️ WEEK 9: System Design — HLD
**Theme**: Scaling Backend Systems for 10M+ Users

### Core HLD Concepts:

#### Caching Strategy
| Cache Type | Use Case | Tool |
|------------|----------|------|
| CDN Cache | Static assets | CloudFront, Fastly |
| Application Cache | Session data | Redis, Memcached |
| Write-through | Data consistency priority | Redis + DB |
| Cache-aside (Lazy) | Read-heavy workloads | Redis + App logic |

#### Kafka vs RabbitMQ
| Feature | Kafka | RabbitMQ |
|---------|-------|----------|
| Use Case | Event streaming | Task queues |
| Retention | Log-based (days/weeks) | ACK-based |
| Best for | Audit logs, Analytics | Email, Background jobs |

#### Database Sharding Strategies
```
Range Sharding     → User IDs 1-1M → Shard A (hotspot risk)
Hash Sharding      → hash(user_id) % N = shard (even distribution)
Consistent Hashing → Ring-based (sabse preferred!)
```

### LRU Cache (Node.js) — Must Know:
```javascript
class LRUCache {
  constructor(capacity) {
    this.capacity = capacity;
    this.map = new Map(); // Key → Value
  }

  get(key) {
    if (!this.map.has(key)) return -1;
    const val = this.map.get(key);
    this.map.delete(key);   // Remove from current position
    this.map.set(key, val); // Re-insert at end (most recently used)
    return val;
  }

  put(key, value) {
    if (this.map.has(key)) this.map.delete(key);
    else if (this.map.size >= this.capacity) {
      this.map.delete(this.map.keys().next().value); // Evict LRU
    }
    this.map.set(key, value);
  }
}
// Note: JavaScript Map maintains insertion order!
```

### REST API Design Checklist:
```
✅ Nouns in URLs       → /users, not /getUser
✅ Correct HTTP Methods → GET, POST, PUT, PATCH, DELETE
✅ Proper status codes  → 200, 201, 400, 401, 403, 404, 409, 500
✅ Pagination           → ?page=1&limit=20 or cursor-based
✅ Versioning           → /api/v1/users
✅ Rate Limiting headers → X-RateLimit-Remaining
✅ Idempotency          → PUT/DELETE are idempotent
✅ Error response format → { error: { code, message, details } }
```
